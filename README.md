## 1.0 üßê General <a name = "general"></a>
Our project aims to actively engage young users in their communities by providing a space to share local events, perspectives, and participate in discussions effortlessly. We built this initiative to empower youth to stay informed, connect with peers, and make their voices heard in their local areas. Through this project, we learned the significance of technology in fostering meaningful connections and empowering youth to contribute to their communities.

### 1.3 Features <a name = "features"></a>
- <i>**User Authentication**</i>: Secure login and registration process to authenticate users and protect their accounts.
- <i>**Feed**</i>: The feed is a central feature displaying content from users / news channels. It is crucial to implement algorithms for content sorting, fetching, and caching efficiently to ensure a smooth scrolling experience.
- <i>**Photo and Video Upload**</i>: The ability to upload photos and videos from the user's device gallery or directly from the camera. 
- <i>**Live Tab**</i>: Allows users to discover new live content, trending hashtags, and explore accounts based on interests. Implementing personalized recommendations and search functionalities enhances user engagement.
- <i>**Notifications**</i>: Notifications inform users about likes, comments, mentions, and new followers. Implementing push notifications and in-app notifications helps keep users engaged and informed.
- <i>**Profile Management**</i>: Users can view and edit their profiles, including bio, profile picture, and settings. Implementing features like profile customization, privacy settings, and account management enhances user control and personalization.
- <i>**Security and Privacy**</i>: Implementing measures like a verification process, data encryption, and content moderation tools to protect user data and maintain a safe environment for users.

Om het voor jezelf makkelijk te maken is het aan te raden om veel kleine commits te maken. Als je later iets kapot hebt gemaakt wat eerst wel werkte, dan kan je deze code makkelijk terugvinden in je commits.*

Code style:
1. Eslint has been added to the project to ensure consistent code style.
2. If you use double quotes, an error should occur. This is automatically resolved when you save the code.
3. If eslint is not applied correctly, it may be because you have the Prettier extension enabled in VSCode. You should disable it.
4. If this is not the case, download the eslint extension in VSCode.
5. To minimize issues during Pull/Merge Requests, it's helpful to sort your imports in ascending order. You can do this by selecting your imports, then pressing CTRL+Shift+P and clicking on "sort lines ascending".*

*[NL] Code style:
1. Eslint is toegevoegd aan het project om ervoor te zorgen dat de code style gelijk is.
2. Als je dubbele quotes gebruikt moet er een error komen. Dit wordt automatisch opgelost als je de code opslaat.
3. Als de eslint niet goed wordt toegepast kan dit komen doordat je de Prettier extension hebt draaien in VSCode. Deze moet je dan uitzetten.
4. Mocht dit niet het geval zijn, download dan de eslint extension in VSCode.
5. Om minder problemen te hebben tijdens Pull/Merge Request, is het handig om je imports ascending te sorteren. Dit doe je door je imports te selecteren en vervolgens CTRL+Shift+P te klikken en vervolgens op sort lines ascending te klikken.* <br>
